# ✅ 知识库 Review 清单

> 每一份贡献都应经过“自查-同级 Review-归档”三阶段，详细流程请参考 [知识库维护指南](../docs/contribution_guide.md)。

---

## 🕒 阶段一：自查 (Self-Check)
- [ ] **内容准确性**: 技术描述、代码逻辑是否准确无误？
- [ ] **代码可运行**: 是否包含单元测试？`pytest` 或 `npm test` 是否 100% 通过？
- [ ] **链接有效性**: 文档内所有外链、内链是否均可正常跳转 (无 404)？
- [ ] **模板规范**: 是否严格遵循 [标准模板](../docs/template.md)？

## 👥 阶段二：同级 Review (Peer Review)
- [ ] **深度评估**: 技术是否有深度？是否仅仅是 API 的堆砌？
- [ ] **实战价值**: 方案是否在实际项目中具有可复用性？
- [ ] **语言表达**: 逻辑是否清晰？是否存在歧义或错别字？
- [ ] **图示支持**: 复杂流程是否配有 Mermaid 图或截图？

## 📦 阶段三：归档 (Archive)
- [ ] **索引更新**: 是否已在根目录 `README.md` 或子技术栈 `README.md` 中更新索引？
- [ ] **资源导出**: 是否已运行 `tools/gen_anki.py` 同步面试题？
- [ ] **权限检查**: 敏感信息 (API Key, Token) 是否已清理？
