# 🎯 全链路测试策略与计划 (Full-Stack Testing Strategy)

> 本文档定义了项目的质量保障体系，明确了各层级测试的边界、优先级及质量红线。

---

## 1. 测试层级与覆盖范围

| 层级 | 覆盖范围 | 优先级 | 核心目标 | 自动化工具 |
| :--- | :--- | :--- | :--- | :--- |
| **单元测试 (Unit)** | 核心业务逻辑、工具函数、算法 | P0 | 行覆盖率 ≥80%, 分支 ≥75% | Pytest / Jest |
| **集成测试 (Integration)** | API 契约、数据库交互、第三方服务调用 | P1 | 验证服务间通信与异常容错 | Pytest + Requests |
| **端到端测试 (E2E)** | 关键用户旅程 (Happy Path)、核心 UI 交互 | P2 | 模拟真实用户行为，确保业务闭环 | Playwright |

---

## 2. 质量红线 (Quality Gates)

在 MR (Merge Request) 阶段强制执行以下门禁，未达标禁止合并：
1. **通过率**: 自动化测试用例通过率必须为 **100%**。
2. **覆盖率**: 核心模块行覆盖率必须达标。
3. **缺陷限制**: 禁止存在任何 **Critical** 或 **High** 级别的未修复缺陷。
4. **性能指标**: 核心接口响应时间 (P95) 不得超过预设阈值。

---

## 3. 测试设计规范 (Given-When-Then)

所有测试用例必须遵循标准结构：
- **Given**: 准备测试数据与环境（使用 Fixtures）。
- **When**: 执行被测操作。
- **Then**: 验证结果与断言。

---

## 4. 自动化回归策略
- **持续集成**: 每次 Push/MR 自动触发全量 Unit 与 API 测试。
- **定时回归**: 每日凌晨 02:00 自动运行 E2E 关键链路测试。
- **报告汇总**: 自动生成 Allure 报告，汇总通过率、覆盖率及趋势图。
