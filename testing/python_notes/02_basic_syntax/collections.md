## 概念
- 列表适合有序可变数据，元组适合不可变记录，集合适合去重与集合运算，字典适合键值映射。
- 选择容器类型时，先想数据是否需要保持顺序、是否允许重复、是否需要按键快速查找。
- 推导式让常见转换与过滤操作更紧凑，但表达式过长时仍然应该拆开写。

## 语法规则
- 列表常用 `append`、`extend`、`pop`；字典常用 `get`、`update`、`items`。
- 元组一旦创建不可修改，适合表达“不会变”的结构化数据。
- 集合天然去重，支持交集 `&`、并集 `|`、差集 `-`。
- 字典基于哈希表实现，按键查找通常是 O(1) 平均复杂度。

## 代码示例
### 示例 1：列表与推导式

```python hl_lines="2 3"
raw_scores = [60, 72, 88, 91]
passed_scores = [score for score in raw_scores if score >= 60]
doubled = [score * 2 for score in raw_scores]

print(passed_scores)
print(doubled)
```


### 示例 2：集合去重与交集

```python hl_lines="4 5"
api_cases = {"login", "logout", "order"}
ui_cases = {"login", "dashboard"}

print(api_cases & ui_cases)
print(api_cases | ui_cases)
```


### 示例 3：字典增删改查

```python hl_lines="2 3 4"
report = {"passed": 18, "failed": 2}
report["skipped"] = 1
report["failed"] += 1
failed = report.get("failed", 0)

print(report)
print(failed)
```

## 易错点
- 把默认参数写成空列表或空字典，会让多个调用共享同一份可变对象。
- 误以为集合有稳定顺序，结果在断言展示或导出时得到不一致顺序。
- 在推导式里塞太多逻辑会让代码变短但更难读，复杂条件宁可拆成普通循环。

## 参考答案
### 概念题 1
当你更关心“是否存在”和“去重”而不是顺序时，应优先使用集合，因为查找和集合运算更自然也更高效。
讲解回看: [概念](#概念)

### 概念题 2
因为它能通过键快速累加状态、场景名或接口名对应的计数，写法直观，查找成本也低。
讲解回看: [语法规则](#语法规则)

### 概念题 3
元组不可变，更适合表达固定结构；列表可变，更适合追加、删除和批量处理。
讲解回看: [语法规则](#语法规则)

### 编程题 1
把它们转成集合后做 `set_a & set_b`。这比手工双重循环更简洁也更高效。
讲解回看: [代码示例](#代码示例)

### 编程题 2
使用 `dict.get(key, default)`。这样即使键不存在也不会抛 `KeyError`。
讲解回看: [代码示例](#代码示例)

可用字典做计数，必要时配合集合记录已见标签。字典负责累加，集合负责快速成员检查，职责清晰。
